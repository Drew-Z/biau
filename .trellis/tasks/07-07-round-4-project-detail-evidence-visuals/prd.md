# Project detail evidence and visuals

## Goal

继续把项目详情页做成访客可读的技术案例页：内容要有实现、架构、流程、质量证据、当前不足、后续优化，并在正文中穿插截图、流程图或状态图，而不是只靠顶部单图。

## Requirements

- 从真实项目文件、脚本、部署状态和已有截图中取证，不只看 README。
- 优先处理内容较薄、配图不准确或状态表达不清的项目。
- 图示必须公开安全，不包含私有后台、token、数据库 URL、真实用户数据。
- 更新项目内容后同步助手知识和必要 sitemap/status。

## Acceptance Criteria

- [ ] 至少一个项目详情页的证据、图示或内容边界有实质提升。
- [ ] 更新内容和真实项目状态一致，不夸大未完成能力。
- [ ] 运行 `assistant:index`、必要内容检查、`lint`/`build` 或 UI 检查。
- [ ] 后续优化方向保留为可执行版本迭代项。
