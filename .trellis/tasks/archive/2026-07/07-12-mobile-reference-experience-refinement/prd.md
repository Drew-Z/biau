# Mobile reference experience refinement

## Goal

吸收参考站在手机端的编辑式信息节奏与阅读清晰度，在不恢复横向手势、不增加底部层级冲突的前提下，让首页、项目目录、博客目录和两类详情页更紧凑、可读、易于单手浏览。

## Background

真实 390×844 截图和几何审计确认：

- 首页首个项目入口约在 `y=452`，首屏品牌表达清楚但有效入口偏晚。
- 项目页首个分组入口约在 `y=325`，项目英雄区和分组间距仍可压缩。
- 博客页首篇文章入口约在 `y=464`，栏目筛选与搜索分别占用两个独立表面，工具区纵向成本最高。
- 博客详情长标题在 390px 上达到五行，英雄卡占据约半屏；内容可读但标题尺度压过摘要和正文入口。
- 当前所有审计路由横向溢出均为 0，底部导航、整卡入口、自然纵向滚动和流体背景均应保持。
- 参考站值得吸收的是有限行长、紧凑移动页头、44px 触控目标、工具聚合和正文去卡片化；其底部目录抽屉不适合直接照搬，因为主站底部已有全局导航和公开助手。

## Requirements

- R1：压缩手机目录页的英雄区、区块间距和工具区垂直占用，但保留品牌识别与清晰标题。
- R2：博客栏目选择、搜索和结果摘要在手机端形成一个连续的工具表面，不再表现为多张相邻浮卡。
- R3：项目分组切换保持一次只展示一组，但减少分组表面和项目卡的套卡感；按钮和整卡入口继续满足 44px 触控目标。
- R4：首页在不删除品牌主标题的情况下，让首个项目入口更早进入首屏，并保留当前纵向自然滚动。
- R5：博客和项目详情的移动标题使用稳定的固定断点字号与行高，不按 viewport 连续缩放；超长标题不得挤压摘要、元数据和阅读导航。
- R6：详情正文继续使用完整页面 document scrolling；不恢复横向滑动，不把阅读目录迁移到底部抽屉。
- R7：背景视觉强度不得降低到“太素”；同时前景文字、边框和表面必须在三场景、明暗主题下保持可读。
- R8：保留底部导航、公开助手互斥、阅读导航自动显隐、safe-area、reduced-motion 和现有桌面布局。

## Acceptance Criteria

- 390×844 下：首页首个项目入口顶部不晚于 410px；项目页首个分组入口不晚于 300px；博客首篇文章入口不晚于 410px。
- 320/390/430px 下博客工具区只有一个连续视觉表面，栏目、搜索、结果摘要均可操作且无文字溢出。
- 320/390/430px 下项目分组和项目卡不出现双层同强度边框/阴影，触控目标均不小于 44px。
- 320px 下最长公开博客标题和项目详情标题可完整换行，无横向溢出；标题、摘要、元数据之间无重叠。
- 首页、项目、博客、博客详情、项目详情在 320/390/430px 均无横向溢出，底部导航不遮挡最终内容。
- 桌面关键布局与现有导航结构保持不变。
- 浅色/深色及 dusk/garden/stellar 场景截图中文字对比与表面层级清楚。
- `npm.cmd run lint`、`npm.cmd run build`、`npm.cmd run performance:check`、`npm.cmd run check:ui` 通过。

## Out of Scope

- 不重新设计桌面版。
- 不引入新的横向手势、scroll-snap、GSAP 或 UI 框架。
- 不更改博客或项目内容数据。
- 不重新设计公开助手或底部导航信息架构。
