# Project Inventory

请盘点当前展示系统中的项目资料，并输出可落地的内容更新建议。

步骤：

1. 先运行 `git status --short --branch`，确认工作区状态。
2. 阅读 `src/data/portfolio.ts`、`src/App.tsx`、`src/App.css`。
3. 阅读 `../reference-projects` 中与当前展示项目相关的 README、文档和目录结构。
4. 不要修改 `../reference-projects`。
5. 不要收录或扩展 `douyu`、`yihuan-helper`、`ques`。
6. 对每个项目输出：定位、技术栈、核心功能、工程结构、实现方式、可展示亮点、脱敏风险。
7. 区分“项目详情页”和“案例详情页”的内容侧重点。
8. 最后给出建议修改清单，先不要大改代码，除非用户明确要求继续实现。
